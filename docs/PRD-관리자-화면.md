# PRD — 관리자 화면

> 관련 문서: [프로젝트 PRD](./PRD.md) · [영화 리뷰 화면 PRD](./PRD-영화-리뷰-화면.md)

## 1. 화면 개요

### 1.1 화면명

관리자 페이지

### 1.2 목적

영화 메타데이터(제목, 장르, 개봉일, 포스터)를 등록하고, 등록된 영화 목록을 조회·수정·삭제하는 관리용 화면이다. 로그인 없이 접근 가능하나, URL/네비게이션으로만 구분하는 단순 관리 UI이다.

### 1.3 접근 경로

- 공통 헤더의 **관리자** 네비게이션 선택 시

### 1.4 와이어프레임

*(와이어프레임 이미지: 우측 「관리자 페이지」 화면)*

## 2. 레이아웃 구조

| 영역 | 설명 |
|------|------|
| 헤더 | 페이지 타이틀, 아이콘, 글로벌 네비게이션 |
| 영화 추가 폼 | 신규 영화 입력 필드 + 추가 버튼 |
| 등록된 영화 목록 | 테이블 형태 목록 + 행별 관리 버튼 |
| 페이지네이션 | 테이블 목록 페이지 이동 |

전체 콘텐츠는 중앙 정렬된 단일 컬럼 레이아웃을 사용한다.

## 3. UI 구성 요소

### 3.1 헤더

| 요소 | 요구 사항 |
|------|-----------|
| 아이콘 | 톱니(설정) 형태의 관리자 페이지 아이콘 |
| 타이틀 | **관리자 페이지** 텍스트 표시 |
| 네비게이션 | **영화 리뷰**, **관리자** — 우측 배치 |
| 활성 상태 | 현재 화면인 **관리자**에 밑줄(또는 동등한 활성 스타일) 표시 |

**동작**

- **영화 리뷰** 클릭 시 영화 리뷰(메인) 화면으로 이동한다.

### 3.2 영화 추가하기 (폼 섹션)

섹션 제목: **영화 추가하기**

| 필드 | 라벨 | 입력 타입 | placeholder / 형식 | 필수 |
|------|------|-----------|----------------------|------|
| 제목 | 제목 | text | `영화 제목을 입력하세요` | O |
| 장르 | 장르 | text | `예) SF, 액션, 드라마 등` | O |
| 개봉일 | 개봉일 | date | `YYYY-MM-DD`, 달력 UI(브라우저 date picker 허용) | O |
| 포스터 URL | 포스터 URL | url 또는 text | `포스터 이미지 URL을 입력하세요` | 선택(빈 값 시 플레이스홀더 이미지) |

| 요소 | 요구 사항 |
|------|-----------|
| 추가 버튼 | 라벨 **영화 추가**, 폼 하단 중앙, 주요 액션 색(파란색), 가로 폭 넓게(full-width 스타일 가능) |

**유효성 검사**

- 제목·장르·개봉일 미입력 시 제출 불가, 필드 근처 또는 상단에 오류 메시지 표시.
- 개봉일: 유효한 날짜 형식만 허용.
- 포스터 URL: URL 형식이 아니면 경고(선택 필드는 저장 시 무시 또는 오류 — 구현 시 **경고 후 저장 허용** 권장).
- 동일 제목 중복 등록 시 서버 오류 메시지를 사용자에게 표시.

**동작**

- **영화 추가** 성공 시: 폼 초기화, 하단 「등록된 영화 목록」 테이블에 신규 행 반영(1페이지 맨 위 등 정책은 구현 시 통일).
- 실패 시: 오류 메시지 표시, 입력값 유지.

### 3.3 등록된 영화 목록 (테이블 섹션)

섹션 제목: **등록된 영화 목록**

| 컬럼 | 설명 |
|------|------|
| 포스터 | 작은 썸네일. `poster_url` 이미지 또는 기본 플레이스홀더 |
| 제목 | 영화 제목 텍스트 |
| 장르 | 장르 텍스트 |
| 개봉일 | `YYYY-MM-DD` |
| 관리 | 행 단위 **수정**, **삭제** 버튼 |

**빈 상태**

- 등록 영화가 없으면 「등록된 영화가 없습니다」 등 안내 문구를 테이블 영역에 표시한다.

**동작 — 수정**

- **수정** 클릭 시 해당 행 데이터로 편집 UI 진입.  
  - MVP: 인라인 편집, 모달 폼, 또는 상단 「영화 추가하기」 폼에 값을 채워 수정 모드 전환 중 하나.  
  - 저장 시 테이블 행 갱신, 영화 리뷰 화면 목록과 동기화(API 반영).

**동작 — 삭제**

- **삭제** 클릭 시 확인 대화상자(예: 「이 영화를 삭제할까요? 연결된 리뷰도 삭제됩니다」).  
  - 확인 후 API 삭제, 성공 시 해당 행 제거.  
  - 연관 리뷰 CASCADE 삭제는 백엔드 정책에 따름.

### 3.4 페이지네이션

| 요소 | 요구 사항 |
|------|-----------|
| 위치 | 테이블 하단 중앙 |
| 구성 | 페이지 번호(예: 1, 2) + 다음 `>` 버튼 |
| 활성 페이지 | 현재 페이지 강조(파란색 배경) |

**동작**

- 페이지당 행 수는 상수로 정의(예: 10개). 추가·삭제·수정 후 현재 페이지 목록을 다시 불러온다.

## 4. 사용자 시나리오

| # | 시나리오 | 기대 결과 |
|---|----------|-----------|
| 1 | 관리자 화면 진입 | 추가 폼과 등록 영화 테이블이 표시된다 |
| 2 | 영화 추가 | 필수 항목 입력 후 **영화 추가** → 테이블에 새 영화가 나타난다 |
| 3 | 영화 수정 | **수정** → 값 변경 저장 → 테이블·메인 화면 데이터 일치 |
| 4 | 영화 삭제 | **삭제** → 확인 → 행 제거, 메인 목록에서도 사라짐 |
| 5 | 페이지 이동 | 등록 영화가 많을 때 다른 페이지 행 조회 |
| 6 | 메인으로 복귀 | **영화 리뷰**로 사용자용 목록 화면 이동 |

## 5. 데이터·API 연동 (참고)

### 5.1 영화 엔티티 필드

| 필드 | 타입 | 설명 |
|------|------|------|
| `id` | number | 식별자 |
| `title` | string | 제목, UNIQUE 권장 |
| `genre` | string | 장르 |
| `release_date` | date (ISO date) | 개봉일 |
| `poster_url` | string \| null | 포스터 이미지 URL |

### 5.2 API (예시)

| 메서드 | 경로 | 용도 |
|--------|------|------|
| GET | `/api/movies` | 목록(페이지네이션·검색 쿼리 optional) |
| POST | `/api/movies` | 영화 추가 |
| PUT | `/api/movies/:id` | 영화 수정 |
| DELETE | `/api/movies/:id` | 영화 삭제 |

리뷰 CRUD는 본 화면 범위 외 — 영화 리뷰 화면에서 처리.

## 6. 비기능·UI 가이드

- **디자인**: 영화 리뷰 화면과 동일한 헤더·색상·버튼 스타일 유지
- **일관성**: 메인 화면 헤더는 「영화 리뷰 앱」, 관리자 화면 헤더는 「관리자 페이지」로 와이어프레임과 동일하게 구분
- **보안**: 로그인 없음 — 프로덕션 배포 시 관리자 URL 노출에 대한 인지 필요(본 PRD 범위에서는 인증 제외)

---

## 7. 범위 외 (본 화면)

- 리뷰·별점 작성·수정·삭제 → **영화 리뷰** 화면
- 사용자 계정·권한 관리
- 포스터 파일 업로드(와이어프레임은 URL 입력만 명시)

---

## 8. 수용 기준 (Acceptance Criteria)

- [ ] 헤더에 **관리자 페이지** 타이틀·아이콘·네비가 있고 **관리자**가 활성 표시된다
- [ ] **영화 추가하기** 폼에 제목·장르·개봉일·포스터 URL 필드와 **영화 추가** 버튼이 있다
- [ ] 필수값 검증 후 영화 등록이 되고 테이블에 표시된다
- [ ] **등록된 영화 목록** 테이블에 포스터·제목·장르·개봉일·관리(수정/삭제)가 있다
- [ ] **수정**으로 영화 정보 변경이 가능하다
- [ ] **삭제**로 확인 후 영화가 제거된다
- [ ] 목록이 길 때 페이지네이션이 동작한다
- [ ] **영화 리뷰** 링크로 메인 화면 이동이 가능하다
