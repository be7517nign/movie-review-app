# PRD — 프런트엔드 UI (개요)

> 프로젝트 전체: [PRD.md](./PRD.md)

## 1. 앱 구조

영화 리뷰 앱 프런트엔드는 **2개의 화면**으로 구성한다.

| 화면 | 문서 | 역할 |
|------|------|------|
| 영화 리뷰 | [PRD-영화-리뷰-화면.md](./PRD-영화-리뷰-화면.md) | 영화 검색·목록·리뷰 열람·리뷰 작성 |
| 관리자 | [PRD-관리자-화면.md](./PRD-관리자-화면.md) | 영화 메타데이터 등록·수정·삭제 |

## 2. 공통 UI

- **글로벌 네비게이션**: 모든 화면 헤더 우측에 `영화 리뷰` · `관리자` 링크
- **활성 표시**: 현재 화면 링크에 밑줄 등 강조 스타일
- **디자인 원칙**: 간단한 레이아웃, 흰/회색 배경, 파란색 주요 버튼·활성 페이지
- **인증**: 없음 (PRD 기본 사항)

## 3. 화면별 책임 분리

```mermaid
flowchart LR
  subgraph user [영화 리뷰 화면]
    A[검색]
    B[영화 카드 목록]
    C[리뷰 작성]
  end
  subgraph admin [관리자 화면]
    D[영화 추가 폼]
    E[영화 테이블 CRUD]
  end
  admin -->|영화 데이터| user
```

- **관리자**: 영화(`title`, `genre`, `release_date`, `poster_url`) 관리
- **영화 리뷰**: 영화 조회·검색·리뷰/별점 작성

## 4. 와이어프레임

통합 와이어프레임(좌: 영화 리뷰, 우: 관리자)은 프로젝트 `assets` 폴더의 와이어프레임 이미지를 참고한다.

각 화면 PRD의 「와이어프레임」 절에서 상세 UI 요구를 정의한다.
